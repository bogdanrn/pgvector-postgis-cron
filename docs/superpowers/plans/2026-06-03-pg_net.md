# pg_net Async HTTP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add async HTTP-from-Postgres to the pg18 image by building `pg_net` v0.20.3 from source in a multi-stage Dockerfile.

**Architecture:** A `build` stage installs compilers + dev headers and compiles pg_net; the final stage installs the existing apt extensions plus the `libcurl4` runtime and `COPY`s the three pg_net artifacts from the build stage. No compilers ship in the final image.

**Tech Stack:** Docker multi-stage (buildx), Debian Trixie, PostgreSQL 18, pg_net (libcurl), pg_cron.

### Task 1: Multi-stage Dockerfile with pg_net build

**Files:**
- Modify: `Dockerfile` (full rewrite)

- [ ] **Step 1: Write the new Dockerfile**

Replace the entire contents of `Dockerfile` with:

```dockerfile
FROM pgvector/pgvector:pg18-trixie AS build

ARG PG_NET_VERSION=0.20.3

RUN apt-get update \
  && apt-get install -y --no-install-recommends \
    build-essential \
    git \
    ca-certificates \
    postgresql-server-dev-18 \
    libcurl4-openssl-dev \
    libicu-dev \
  && git clone --depth 1 --branch v${PG_NET_VERSION} \
    https://github.com/supabase/pg_net.git /tmp/pg_net \
  && make -C /tmp/pg_net -j"$(nproc)" \
  && make -C /tmp/pg_net install

FROM pgvector/pgvector:pg18-trixie

RUN apt-get update \
  && apt-get install -y --no-install-recommends \
    postgresql-contrib \
    postgresql-18-postgis-3 \
    postgresql-18-postgis-3-scripts \
    postgresql-18-cron \
    libcurl4 \
  && rm -rf /var/lib/apt/lists/*

COPY --from=build /usr/lib/postgresql/18/lib/pg_net.so \
  /usr/lib/postgresql/18/lib/
COPY --from=build /usr/share/postgresql/18/extension/pg_net* \
  /usr/share/postgresql/18/extension/
```

### Task 2: Runtime smoke test — extension loads and a request resolves

**Files:** none (verification task; no commit unless a fix to Task 1 is needed)

- [ ] **Step 1: Start a container with pg_net preloaded**

Run:
```bash
docker run -d --name pgnet-smoke \
  -e POSTGRES_HOST_AUTH_METHOD=trust \
  pgvector-postgis-cron:test \
  postgres \
    -c shared_preload_libraries=pg_stat_statements,pg_cron,pg_net \
    -c cron.database_name=postgres
```
Expected: prints a container id.

- [ ] **Step 2: Wait for readiness**

Run:
```bash
until docker exec pgnet-smoke pg_isready -U postgres >/dev/null 2>&1; do sleep 1; done; echo READY
```
Expected: `READY` within a few seconds.

- [ ] **Step 3: Create the extension**

Run:
```bash
docker exec pgnet-smoke psql -U postgres -c "CREATE EXTENSION pg_net;"
```
Expected: `CREATE EXTENSION`.

- [ ] **Step 4: Fire an async request and confirm it resolves into `net._http_response`**

Run:
```bash
docker exec pgnet-smoke psql -U postgres -c \
  "SELECT net.http_get('https://example.com');"
sleep 5
docker exec pgnet-smoke psql -U postgres -tAc \
  "SELECT status_code FROM net._http_response ORDER BY id DESC LIMIT 1;"
```
Expected: first command returns a request id (integer); second returns `200`.

- [ ] **Step 5: Tear down**

Run:
```bash
docker rm -f pgnet-smoke
```
Expected: `pgnet-smoke`.

> If Step 3 or 4 fails, the defect is in the Dockerfile (Task 1) — fix there,
> rebuild, and re-run this task. Do not patch around it here.
